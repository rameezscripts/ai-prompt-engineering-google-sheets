Step 01

Why Your Prompt is Everything

AI gives bad code when you give it bad prompts. The output quality is 100% based on how you ask. More details = better code. Less details = garbage code.

Think of AI like a new employee — it knows nothing about your sheet. Your job is to tell it everything: sheet name, columns, what to do, what not to do.
Step 02

Bad Prompt vs Good Prompt

This is the most important thing to understand. See the difference between a lazy prompt and a proper one.

Bad Prompt ChatGPT 
Make a Google Sheet script that sends emails
Good Prompt ChatGPT 
Write a Google Apps Script function that reads
from a sheet named "Employees" and sends emails.

Sheet columns:
- A: Employee Name
- B: Email Address
- C: Department
- D: Status ("Active" or "Inactive")

Rules:
- Only send to Status = "Active" employees
- Subject: "Monthly Update - [Department]"
- Body: include employee name
- Skip empty rows
- After sending, write "Sent" in column E with today's date
- Use GmailApp.sendEmail()

Return only the function. Short comments only.
Step 03

The 5 Pillars of a Perfect Prompt

Every good code prompt must have these 5 things. Missing even one = weak output.

01

Context

Sheet name, column layout, what the project is about.

02

Task

Exactly what you want — "write a function that...", "fix this bug".

03

Constraints

What NOT to do — "no triggers", "manual run only", "1000+ rows".

04

Output Format

"Code only", "add short comments", "return as JSON".

05

One Example

Show one row of data: "Name=Ali, Status=Active → send email".

Step 04

ChatGPT — Numbered Rules Work Best

Style: Numbered list, clear rules, like a spec document

Task: write a function to read active employees from a sheet.

ChatGPT Prompt ChatGPT 
Write a Google Apps Script function called getActiveEmployees
that reads from a sheet and returns active employees.

Sheet name: "Employees"
Columns: A=ID, B=Name, C=Email, D=Department, E=Status

Rules:
1. Skip header row (row 1)
2. Read all data in ONE getValues() call — no cell-by-cell
3. Filter: only rows where Status = "Active"
4. Skip empty rows (where column B is empty)
5. Return: array of objects [{id, name, email, department}]

Code only. Short inline comments.
Step 05

Gemini — Context First, Task at the End

Style: Full background first → question at the end

Same task, different style. Gemini performs best when you describe everything upfront.

Gemini Prompt Gemini 
Context:
I have a Google Sheet called "Employees" with columns:
A=ID, B=Name, C=Email, D=Department, E=Status (Active/Inactive).
About 500 rows. I'm building a web app with Apps Script HTML Service.

Task:
Write a function called getActiveEmployees that:
- Reads all data from the "Employees" sheet
- Filters only Status = "Active" rows
- Skips header row and empty rows
- Returns array of objects: [{id, name, email, department}]

Performance: read all data in one getValues() call.
Code only. Short developer-style comments.
Step 06

Claude — XML Tags for Each Section

Style: <context> <task> <rules> <output> tags

Claude understands XML structure very well — it reads each tag as a separate instruction block.

Claude Prompt Claude 
<context>
Google Apps Script. Sheet: "Employees"
Columns: A=ID, B=Name, C=Email, D=Department, E=Status
Values: "Active" or "Inactive". ~500 rows.
</context>

<task>
Write function getActiveEmployees — read sheet, return only active employees.
</task>

<rules>
- One getValues() batch call only
- Skip header row (row 1)
- Skip empty rows (column B empty)
- Filter: column E = "Active"
- Return: [{id, name, email, department}]
</rules>

<output>
Code only. Short inline comments. No explanation.
</output>
Step 07

DeepSeek — Explicit Steps with Column Indexes

Style: Step-by-step with exact column indexes and data types

DeepSeek loves precise, technical instructions. Give it exact indexes and it follows them perfectly.

DeepSeek Prompt DeepSeek 
Write Google Apps Script function: getActiveEmployees()

Sheet: "Employees"
Column map: A(0)=ID, B(1)=Name, C(2)=Email, D(3)=Dept, E(4)=Status

Steps:
1. getSheetByName("Employees")
2. getDataRange().getValues() — single batch call
3. data.shift() — remove header row
4. filter: row[4] === "Active"
5. filter: row[1] !== "" — skip empty rows
6. map: {id:row[0], name:row[1], email:row[2], department:row[3]}
7. return the array

Code only. Short comments.
Key Point: All 4 prompts ask for the same thing — just styled differently per AI. ChatGPT = numbered rules. Gemini = context first. Claude = XML tags. DeepSeek = exact indexes. Match the style and you get better results with less back-and-forth.
Step 08

How to Fix a Bug with AI

Always give AI 3 things: the code, what it should do, and what it's actually doing wrong.

Bug Fix Prompt Gemini 
My Apps Script function should send emails only to "Active"
employees, but it's sending to ALL employees.

Code:
```javascript
function sendEmails() {
  var sheet = SpreadsheetApp.getActiveSpreadsheet()
    .getSheetByName("Employees");
  var data = sheet.getDataRange().getValues();

  for (var i = 1; i < data.length; i++) {
    var name = data[i][1];
    var email = data[i][2];
    var status = data[i][4];

    if (status = "Active") {         // <-- problem here
      GmailApp.sendEmail(email, "Monthly Update",
        "Hello " + name + ", here is your update.");
    }
  }
}
```

Columns: A=ID, B=Name, C=Email, D=Dept, E=Status
Expected: send only to Status = "Active"
Actual: sending to everyone

Find bug. One line explanation. Show fixed code.
Step 09

How to Get the Perfect Sheets Formula

Tell AI your column layout and exactly what calculation you need. The more specific, the better the formula.

Formula Prompt DeepSeek 
Write a Google Sheets formula for cell F2.

Sheet "Sales" columns: A=Date, B=Salesperson, C=Product, D=Qty, E=Unit Price, F=Total

What I need:
- F2 = D2 * E2
- Use ARRAYFORMULA so it auto-fills all rows
- Skip row 1 (header)
- If D or E is empty, show blank — not zero

Formula only. No explanation.
Step 10

5 Golden Rules — Always Remember These

Follow these 5 rules every time you prompt AI for Apps Script or Sheets work.

R1

Always Say "Apps Script"

Not just "JavaScript". Say "Google Apps Script" so AI uses SpreadsheetApp, GmailApp — not Node.js APIs.

R2

Give Sheet Name + Column Map

"Sheet: Employees, A=ID, B=Name, C=Email, D=Status". Without this, AI invents wrong column names.

R3

Show One Row Example

"Row: Name=Ali, Status=Active → send email." One example beats three paragraphs of explanation.

R4

Control the Output

"Code only", "short comments", "formula only". Without this, AI writes paragraphs you don't need.

R5

Never Start a New Chat

If the answer is 80% right, fix it in the same chat. Say "change the return format" — don't restart.
